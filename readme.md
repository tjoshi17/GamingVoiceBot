
# Gaming Analytics Voice Bot

A lightweight Streamlit application that lets you ask voice queries about a gaming analytics SQLite dataset. The app transcribes speech (Whisper), finds a matching SQL query from a stored repository (Chroma vector DB + Ollama embeddings), executes the SQL against `database/gaming.db`, and displays the results.

## Features

- **Voice-powered queries:** Record a question with your microphone and get results back from the DB.
- **Query repository:** Pre-populated SQL queries stored in `query_repository` (SQLite) and indexed in a Chromadb vector store for semantic matching.
- **Local transcription & embeddings:** Uses OpenAI Whisper for transcription and Ollama for embeddings/reranking (local models supported).
- **Lightweight storage:** Uses SQLite for analytics data and Chromadb persistent storage for vectors.

## Requirements

- Python 3.10+ (recommended)
- System dependencies: `ffmpeg` (required by Whisper)
- Python packages (install with `pip install -r requirements.txt`):

See [requirements.txt](requirements.txt) — notable packages include `streamlit`, `openai-whisper`, `ollama`, `chromadb`, `pandas`, and `sqlalchemy`.

## Quick Start

1. Create and activate a virtual environment:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

2. Initialize the SQLite schema:

```bash
python create_tables.py
```

3. Generate sample data using scripts in the `data_generation/` folder.
```bash
python generate_games.py
python generate_players.py
python generate_sessions.py
python generate_transactions.py
python generate_achievements.py
```

4. Load the predefined analytical queries into the database:

```bash
python load_queries.py
```

5. Build the vector index (requires Ollama daemon if using local Ollama embeddings):

```bash
python create_vector_store.py
```

6. Run the Streamlit app:

```bash
streamlit run app.py
```

Open the URL Streamlit prints (usually http://localhost:8501) and use the microphone controls to ask a query.

You can ask questions such as 
    1. How many players exist?
    2. How many VIP players?
    3. Average player level?
    4. Players registered this year?
    5. Total gaming Sessions?
    6. Average session duration?
    7. Top 10 players by playtime?
    8. Daily active users?
    9. Average sessions per player?
    10. Players with more than 100 hours playtime?
    11. Total revenue generated?
    12. Average transaction amount?
    13. Revenue generated this year?
    14. Revenue by game?
    15. Total number of games?
    16. Most played game?
    17. Game with highest revenue?
    18. Revenue amount per game?
    19. Total achievements unlocked?
    20. Most common achievement?
    21. Player with most achievements?
    22. Players with more than 100 hours playtime?
    23. Revenue generated by VIP players?
    24. Revenue generated by VIP players?
    25. Average playtime by game?

If you want any more queries to be included you can add your query to query library by load_queries.py

## Important Files and Scripts

- [app.py](app.py): Streamlit frontend. Records audio, transcribes with Whisper, retrieves best-matching repository query, executes SQL, and displays results.
- [create_tables.py](create_tables.py): Creates `database/gaming.db` and tables (`players`, `games`, `sessions`, `transactions`, `achievements`, `leaderboards`, `query_repository`).
- [load_queries.py](load_queries.py): Inserts many pre-written SQL queries into the `query_repository` table.
- [create_vector_store.py](create_vector_store.py): Reads queries from the DB, generates embeddings (Ollama), and stores them in a Chromadb collection (`vector_db`).
- [requirements.txt](requirements.txt): Python dependencies.

Services (helper modules):

- [services/speech_to_text.py](services/speech_to_text.py): Loads Whisper model and transcribes audio.
- [services/embeddings.py](services/embeddings.py): Wrapper around Ollama embeddings used by the retriever.
- [services/retrieval.py](services/retrieval.py): Queries Chromadb to find top candidate questions.
- [services/reranker.py](services/reranker.py): Optional Ollama-based reranker to pick the best candidate.
- [services/sql_executor.py](services/sql_executor.py): Executes SQL against the SQLite DB and returns a pandas DataFrame.

Data & storage:

- `database/gaming.db`: Primary SQLite database (created by `create_tables.py`).
- `vector_db/`: Chromadb persistent vector store (created by `create_vector_store.py`).

## Notes & Troubleshooting

- Whisper requires `ffmpeg` and a working PyTorch installation. On macOS install with Homebrew: `brew install ffmpeg`.
- Ollama: If you use Ollama embeddings/chat, ensure the Ollama daemon is installed and running locally. Replace with another embedding provider if desired.
- If you change the vector store or want to rebuild it, remove the `vector_db/` directory and run `python create_vector_store.py` again.
- The app writes temporary audio files to the `temp/` folder (e.g., `temp/audio.wav`).

## Extending the Project

- Add or customize SQL queries in `load_queries.py` (they are stored in `query_repository`).
- Replace embedding/reranking model calls in `services/embeddings.py` and `services/reranker.py` to point at different providers (OpenAI, Hugging Face, etc.).
- Swap SQLite for a different production DB by updating `services/sql_executor.py`.
